# MC ERP 后端开发文档

## 项目简介
本项目为 MC ERP 系统后端，基于 Spring Boot 3 + MyBatis-Plus + MySQL，采用分层架构，支持 JWT 鉴权、参数校验、统一异常处理等。

## 技术栈
- Java 17
- Spring Boot 3.1.5
- MyBatis-Plus 3.5.4
- MySQL
- JWT (io.jsonwebtoken)
- EasyExcel (导入导出)
- Lombok

## 目录结构
- `controller/`：接口层，处理 HTTP 请求
- `service/`：业务逻辑层
- `mapper/`：数据访问层，MyBatis-Plus Mapper
- `entity/`：数据库实体
- `dto/`：数据传输对象（请求参数）
- `vo/`：视图对象（返回数据）

## 典型接口说明（以货代订单为例）

### 货代订单接口 `/api/v1/freight-orders`
- `GET /page`：分页查询
- `GET /{orderId}`：查询详情
- `POST /`：创建订单（body: { order, feeItems })
- `PUT /`：修改订单
- `DELETE /{orderId}`：删除订单（仅草稿）
- `PUT /{orderId}/submit`：提交订单
- `PUT /{orderId}/settle`：结算订单
- `PUT /{orderId}/cancel`：作废订单（body: { cancelReason })
- `GET /{orderId}/fee-items`：获取费用明细
- `POST /{orderId}/fee-items`：保存费用明细
- `DELETE /fee-items/{itemId}`：删除单个费用明细
- `GET /{orderId}/logs`：获取操作日志
- `GET /status-meta`：获取订单状态元数据

### 典型数据结构
- DTO/VO/Entity 均有对应 JavaBean，详见 `dto/`、`vo/`、`entity/` 目录。
- 分页查询统一返回 `PageResult<T>`，普通接口返回 `Result<T>`。

## 参数校验
- 使用 `jakarta.validation` 注解（如 `@NotNull`、`@Size`）+ `@Validated` 实现参数校验。

## 安全与异常
- JWT 鉴权，接口需带 Token。
- 统一异常处理，避免异常堆栈泄露。

## 代码规范
- Controller 只做参数接收和简单校验，业务逻辑在 Service 层。
- Mapper 只做数据库操作。
- Entity/DTO/VO 分层清晰，避免混用。

## 运行与开发
- 配置文件：`src/main/resources/application.yml`
- 启动类：`BackendApplication.java`
- 依赖管理：Maven（见 `pom.xml`）

## 其他
- 支持 Excel 导入导出（EasyExcel）
- 详细接口和字段说明请参考源码注释和各目录下 Java 文件。
