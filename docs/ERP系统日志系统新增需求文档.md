# ERP系统日志系统新增需求文档

# 1. 文档概述

## 1.1 文档目的

本文档明确ERP系统新增日志系统的需求规格、功能边界、非功能要求及接口规范，作为开发、测试、部署及验收的核心依据，确保日志系统能够满足ERP系统的操作追溯、故障排查、安全审计及合规要求，保障系统稳定、安全运行。

## 1.2 文档范围

本文档覆盖ERP系统日志系统的全部新增功能，包括日志采集、日志存储、日志查询、日志分析、权限控制、异常告警等模块，涉及ERP系统各核心业务模块（如采购管理、销售管理、库存管理、财务管理、用户管理等）的日志生成与管理，不包含原有ERP系统业务逻辑的修改。

## 1.3 术语定义

- ERP系统：企业资源计划系统，用于整合企业内部各业务模块，实现资源统筹管理、流程标准化的核心系统。

- 日志系统：新增的用于采集、存储、查询、分析ERP系统各类操作及运行状态的子系统，独立于核心业务模块，可联动核心模块实现日志全生命周期管理。

- 操作日志：记录用户在ERP系统中的各类操作行为（如登录、新增、修改、删除、查询等）的日志信息。

- 系统日志：记录ERP系统自身运行状态（如启动、停止、异常报错、接口调用失败等）的日志信息。

- 审计日志：用于满足合规要求，重点记录敏感操作、权限变更、数据修改等关键行为的日志，可用于事后审计追溯。

- 日志脱敏：对日志中包含的敏感信息（如密码、身份证号、银行卡号等）进行加密或替换处理，避免信息泄露。

# 2. 需求背景与目标

## 2.1 需求背景

当前ERP系统未具备完善的日志管理功能，仅能记录简单的用户登录日志，无法实现对各业务模块操作行为、系统运行状态的全面追溯。当出现数据异常、操作失误、系统故障时，无法快速定位问题根源；同时，随着企业合规要求的提升，需要对敏感操作、数据变更进行全程审计，现有日志能力无法满足审计及安全管理需求，因此需新增一套完整的日志系统。

## 2.2 需求目标

- 全面采集：覆盖ERP系统所有用户操作、系统运行、接口调用等场景，实现日志无遗漏采集。

- 安全存储：确保日志数据的完整性、安全性，支持长期存储，防止日志篡改、丢失。

- 便捷查询：提供多条件组合查询功能，支持快速检索所需日志，提升问题排查效率。

- 审计合规：记录敏感操作及数据变更，满足企业内部审计及行业合规要求，可追溯操作源头。

- 异常告警：对系统异常、违规操作等情况进行实时告警，及时发现安全隐患及系统故障。

- 易于维护：日志系统可独立管理，支持日志清理、备份，不影响ERP核心业务系统性能。

# 3. 功能需求

## 3.1 日志采集功能

### 3.1.1 采集范围

日志系统需采集ERP系统以下三类日志，确保采集无遗漏、无重复：

- 用户操作日志：记录所有用户在ERP系统中的操作行为，包括但不限于：登录（成功/失败）、退出、新增数据、修改数据、删除数据、查询数据、导出数据、权限变更、角色分配等。

- 系统运行日志：记录ERP系统自身运行状态，包括但不限于：系统启动、停止、重启、服务异常、数据库连接异常、接口调用失败、资源占用过高（CPU、内存、磁盘）等。

- 接口日志：记录ERP系统内部接口及与外部系统接口的调用情况，包括接口名称、调用时间、调用方、请求参数、响应结果、调用状态（成功/失败）、异常信息等。

### 3.1.2 采集字段

各类日志需包含以下核心字段，根据日志类型补充扩展字段，确保日志信息完整可追溯：

|日志类型|核心字段|扩展字段|
|---|---|---|
|用户操作日志|日志ID、操作时间、操作用户ID、操作用户名称、操作IP、操作模块、操作类型、操作描述、操作结果（成功/失败）|操作前数据、操作后数据、错误信息（失败时）、操作终端（PC/移动端）|
|系统运行日志|日志ID、日志时间、日志级别（INFO/WARN/ERROR/FATAL）、日志内容、系统模块、运行状态|异常堆栈信息、资源占用情况、服务器IP、进程ID|
|接口日志|日志ID、调用时间、接口URL、调用方ID、请求方法（GET/POST等）、调用状态、响应耗时|请求参数、响应数据、错误码、错误信息、接口所属模块|
### 3.1.3 采集方式

- 用户操作日志：通过AOP切面编程，拦截ERP系统各业务模块的操作接口，自动采集操作信息，无需业务模块单独开发日志生成代码。

- 系统运行日志：集成ERP系统的日志框架（如Logback、Log4j），通过配置日志输出规则，自动采集系统运行状态信息。

- 接口日志：通过接口拦截器，拦截所有内部及外部接口调用，采集接口调用相关信息，支持同步采集，确保日志与接口调用同步生成。

## 3.2 日志存储功能

### 3.2.1 存储介质

采用“本地文件+数据库”双重存储方式，兼顾性能与安全性：

- 本地文件存储：用于存储系统运行日志及接口日志的原始文件，按日期分文件夹存储，便于快速定位及备份，文件格式为.log，编码为UTF-8。

- 数据库存储：用于存储用户操作日志、审计日志及日志索引信息，采用ERP系统现有数据库（如MySQL、Oracle），单独创建日志数据表，与核心业务数据表隔离，避免影响核心业务性能。

### 3.2.2 存储周期

根据日志类型及合规要求，设置不同的存储周期，支持手动调整存储周期配置：

- 用户操作日志：存储周期不少于1年，到期后自动归档至备份服务器。

- 系统运行日志：本地文件存储周期为3个月，数据库存储周期为6个月，到期后自动清理本地文件，数据库日志自动归档。

- 接口日志：存储周期不少于6个月，到期后自动归档。

- 审计日志：存储周期不少于3年，满足合规审计要求，到期后需经管理员确认方可清理或归档。

### 3.2.3 数据安全

- 日志数据不可篡改：数据库日志表设置防篡改机制，对日志数据的修改、删除操作进行记录，且仅超级管理员可执行归档、清理操作。

- 备份机制：每日凌晨自动对日志数据进行备份，备份文件存储在独立的备份服务器，备份保留周期与日志存储周期一致，支持手动备份。

- 权限控制：日志数据的读取、修改、删除权限严格区分，仅授权用户可操作，防止日志数据泄露或被恶意修改。

## 3.3 日志查询功能

### 3.3.1 查询权限

根据用户角色分配不同的日志查询权限，实现权限分级管控：

- 管理员：可查看日志系统

### 3.3.2 查询条件

提供多条件组合查询功能，支持精确查询和模糊查询，查询条件可根据日志类型灵活选择：

- 通用查询条件：日志时间范围（开始时间、结束时间）、日志级别、操作结果（成功/失败）。

- 用户操作日志查询条件：操作用户、操作IP、操作模块、操作类型、操作描述。

- 系统运行日志查询条件：系统模块、日志内容、服务器IP。

- 接口日志查询条件：接口URL、调用方、请求方法、响应耗时、错误码。

### 3.3.3 查询结果展示

- 查询结果以列表形式展示，包含日志核心字段，支持按日志时间（倒序/正序）排序，默认按倒序（最新日志优先）展示。

- 支持分页展示，可自定义每页显示条数（10/20/50/100条）。

- 点击单条日志，可查看日志详情，包括所有字段信息（含扩展字段），系统运行日志需展示完整的异常堆栈信息。

- 支持日志导出功能，导出格式为Excel，可选择导出查询结果的全部数据或选中数据。

## 3.4 日志分析功能

### 3.4.1 操作统计分析

- 按用户统计：统计指定时间范围内各用户的操作次数、操作类型分布（新增/修改/删除等）、操作成功率。

- 按模块统计：统计指定时间范围内各业务模块的操作次数、操作热点（高频操作）、异常操作次数。

- 按时间统计：按小时、天、周、月统计操作总量，生成趋势图，直观展示操作变化情况。

### 3.4.2 系统运行分析

- 异常统计：统计指定时间范围内系统异常次数、异常类型分布（数据库异常、接口异常、资源异常等）、异常发生的服务器及模块。

- 资源监控：结合系统运行日志，分析服务器CPU、内存、磁盘资源的占用趋势，当资源占用超过阈值时，生成预警信息。

- 接口分析：统计接口调用成功率、平均响应耗时、高频调用接口、异常接口，生成接口性能分析报告。

### 3.4.3 审计分析

- 敏感操作统计：统计指定时间范围内的敏感操作（如权限变更、数据删除、敏感数据查询）次数、操作人、操作时间。

- 异常操作追溯：对失败操作、违规操作进行关联分析，定位操作源头，展示操作前后的数据变化，便于审计核查。

- 生成审计报告：支持按指定时间范围生成审计报告，包含敏感操作统计、异常操作统计、合规情况说明，可导出为PDF格式。



## 3.6 日志管理功能

### 3.6.1 日志清理

- 自动清理：根据预设的存储周期，自动清理过期的日志数据（本地文件及数据库日志），清理前自动备份相关日志。

- 手动清理：系统管理员可手动清理指定时间范围、指定类型的日志，清理操作需输入管理员密码确认，清理记录计入审计日志。

### 3.6.2 日志归档

- 自动归档：到期的日志数据（如用户操作日志、审计日志）自动归档至备份服务器，归档文件按日期+日志类型命名，便于查询。

- 手动归档：管理员可手动归档指定时间范围、指定类型的日志，支持归档至本地或备份服务器。

- 归档查询：支持查询归档日志，可按归档时间、日志类型检索，支持下载归档文件。

### 3.6.3 配置管理

- 日志采集配置：支持管理员配置日志采集范围、采集字段、可开启/关闭某类日志的采集功能。

- 存储配置：支持管理员调整日志存储周期、本地文件存储路径、数据库存储参数。

- 权限配置：支持管理员分配日志查询、管理权限，调整不同角色的日志操作权限。

# 4. 非功能需求

## 4.1 性能需求

- 采集性能：日志采集延迟≤1秒，不影响ERP核心业务操作的响应速度（核心业务响应时间仍保持≤3秒）。

- 查询性能：单条件查询响应时间≤1秒，多条件组合查询响应时间≤3秒，分页查询（100条/页）响应时间≤2秒。

- 存储性能：支持日均日志量≤10万条的存储需求，日志写入、读取不占用核心业务数据库的资源（数据库CPU占用率≤10%）。

- 并发性能：支持同时≤50个用户进行日志查询、分析操作，无卡顿、超时现象。

## 4.2 安全性需求

- 权限控制：严格的权限分级机制，不同角色仅能访问授权范围内的日志数据，防止日志泄露。

- 数据防篡改：日志数据一旦生成，不可随意修改、删除，修改/删除操作需记录审计日志，可追溯。

- 备份安全：备份日志存储在独立的备份服务器，具备访问权限控制，防止备份数据丢失或被篡改。

## 4.3 可靠性需求

- 日志采集可靠性：日志采集模块具备容错机制，当ERP系统某模块临时故障时，可缓存日志数据，故障恢复后自动补采，确保日志无遗漏。

- 系统稳定性：日志系统独立运行，与ERP核心业务模块解耦，日志系统故障不影响ERP核心业务的正常运行；日志系统自身可用性≥99.9%。

- 数据完整性：日志数据在采集、存储、传输过程中不丢失、不重复，确保日志信息的完整性。

## 4.4 易用性需求

- 操作界面：日志查询、分析、管理界面简洁直观，与ERP系统现有界面风格一致，操作流程简单，无需专业培训即可上手。

- 查询便捷：支持模糊查询、多条件组合查询，提供查询条件提示，简化查询操作。

- 报表展示：分析结果以图表（柱状图、折线图、饼图）+表格形式展示，直观易懂，支持报表导出。

- 异常提示：操作失败时，给出明确的错误提示，指导用户解决问题；告警信息清晰，包含具体的异常描述及处理建议。

## 4.5 兼容性需求

- 系统兼容性：日志系统需兼容ERP系统现有版本（如ERP V3.0及以上），支持Windows、Linux等操作系统。

- 浏览器兼容性：支持Chrome、Firefox、Edge等主流浏览器（最新版本及前两个版本），界面显示正常，操作无异常。

## 4.6 可扩展性需求

- 功能扩展：日志系统架构设计具备可扩展性，支持后续新增日志类型、新增分析维度、新增告警方式等功能，无需大规模修改现有代码。

- 性能扩展：支持日志存储容量、并发查询量的扩展，可通过增加服务器节点、优化存储策略等方式提升系统性能。

- 接口扩展：提供标准化的日志查询、导出接口，支持与企业现有审计系统、监控系统对接，实现数据互通。

# 5. 接口需求

## 5.1 内部接口

日志系统与ERP核心业务模块的内部接口，用于采集日志数据，接口采用RESTful风格，请求方式为POST，数据格式为JSON：

- 用户操作日志采集接口：接收ERP各业务模块发送的用户操作信息，接口地址：/erp/log/user/collect，请求参数包含操作用户、操作模块、操作类型等核心字段。

- 系统运行日志采集接口：接收ERP系统框架发送的系统运行信息，接口地址：/erp/log/system/collect，请求参数包含日志级别、日志内容、系统模块等核心字段。

- 接口日志采集接口：接收接口拦截器发送的接口调用信息，接口地址：/erp/log/interface/collect，请求参数包含接口URL、调用方、响应结果等核心字段。

## 5.2 外部接口

日志系统对外提供的接口，用于与审计系统、监控系统对接，接口采用RESTful风格，支持GET/POST请求，数据格式为JSON：

- 日志查询接口：供外部系统查询日志数据，接口地址：/erp/log/query，支持多条件查询，返回日志列表及详情。

- 日志导出接口：供外部系统导出日志数据，接口地址：/erp/log/export，支持按条件导出Excel/PDF格式日志。

- 告警推送接口：将日志系统的告警信息推送至外部监控系统，接口地址：/erp/log/alarm/push，推送参数包含告警级别、告警描述、关联日志ID等。

## 5.3 接口要求

- 接口响应时间≤1秒，接口调用成功率≥99.9%。

- 接口需进行身份认证，采用Token验证机制，防止未授权调用。

- 接口支持异常重试机制，当接口调用失败时，自动重试3次，重试间隔1秒。

- 接口返回统一的错误码，包含错误描述，便于问题排查。

# 6. 数据字典

## 6.1 日志级别字典

|字典编码|字典名称|说明|
|---|---|---|
|INFO|提示|系统正常运行信息、用户正常操作信息|
|WARN|警告|系统潜在风险、非致命异常|
|ERROR|错误|系统异常、操作失败、接口调用失败|
|FATAL|致命|系统崩溃、严重故障，影响核心业务运行|
## 6.2 操作类型字典

|字典编码|字典名称|说明|
|---|---|---|
|LOGIN|登录|用户登录系统|
|LOGOUT|退出|用户退出系统|
|ADD|新增|新增数据（如新增采购单、新增用户）|
|MODIFY|修改|修改已有数据（如修改订单信息、修改用户权限）|
|DELETE|删除|删除数据（如删除废弃订单、删除用户）|
|QUERY|查询|查询数据（如查询库存、查询报表）|
|EXPORT|导出|导出数据（如导出报表、导出日志）|
|AUTH_CHANGE|权限变更|修改用户权限、分配角色|
## 6.3 操作结果字典

|字典编码|字典名称|说明|
|---|---|---|
|SUCCESS|成功|操作、接口调用、系统运行正常|
|FAIL|失败|操作、接口调用失败，系统运行异常|
# 7. 验收标准

## 7.1 功能验收

- 日志采集：所有指定范围的日志（用户操作、系统运行、接口）均能正常采集，无遗漏、无重复，敏感信息脱敏正确。

- 日志存储：日志能正常存储至本地文件及数据库，存储周期符合要求，备份、归档功能正常，日志不可篡改。

- 日志查询：不同角色的查询权限正确，多条件组合查询、分页查询、导出功能正常，响应时间符合性能要求。

- 日志分析：操作统计、系统运行分析、审计分析功能正常，图表展示正确，审计报告生成及导出正常。

- 异常告警：告警触发条件正确，告警方式（系统内、邮件、短信）正常，告警状态管理、历史记录查询正常。

- 日志管理：日志清理、归档、配置管理功能正常，操作权限控制正确。

## 7.2 非功能验收

- 性能：采集延迟、查询响应时间、并发性能均符合要求，不影响ERP核心业务性能。

- 安全性：权限控制严格，日志数据防篡改，敏感信息无泄露，备份安全可靠。

- 可靠性：日志采集无遗漏，系统运行稳定，故障时不影响ERP核心业务，数据完整。

- 易用性：操作界面简洁直观，查询、分析、管理操作便捷，错误提示清晰。

- 兼容性：兼容ERP现有版本、数据库及主流浏览器，无显示及操作异常。

## 7.3 接口验收

- 内部接口：日志采集接口能正常接收ERP各模块发送的日志数据，无数据丢失、格式错误。

- 外部接口：日志查询、导出、告警推送接口能正常调用，响应时间、成功率符合要求，身份认证有效。

# 8. 风险与应对措施

|风险类型|风险描述|应对措施|
|---|---|---|
|性能风险|日志采集、存储、查询占用过多系统资源，影响ERP核心业务运行|1. 日志系统与核心业务模块解耦，独立部署；2. 优化日志采集机制，采用异步采集；3. 对日志数据进行分库分表存储；4. 定期清理过期日志，释放资源。|
|数据安全风险|日志数据泄露、被篡改，敏感信息未脱敏|1. 严格执行权限分级控制；2. 日志数据加密存储，防篡改；3. 完善日志脱敏规则，对敏感信息进行严格处理；4. 定期审计日志操作记录。|
|日志遗漏风险|系统故障、接口异常导致日志采集遗漏|1. 日志采集模块增加容错机制，支持日志缓存及补采；2. 定期检查日志采集状态，发现异常及时告警；3. 对关键操作日志进行双重采集验证。|
|兼容性风险|日志系统与ERP现有版本、数据库不兼容|1. 开发前进行兼容性测试，适配现有系统环境；2. 采用标准化的接口及技术架构；3. 预留兼容性扩展接口，便于后续适配升级。|
